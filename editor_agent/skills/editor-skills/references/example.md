# Editor Skill Examples

---

## Hook Examples

Input:
"The Claude code leak was caused by a mistake."

Output:
"No hack. No breach. Just one small mistake… and everything was exposed."

---

Input:
"This article explains a bug."

Output:
"This wasn't a bug. It was a miss."

---

## Clarity Transformations

Input:
"It is important to note that this could have been avoided."

Output:
"This could have been avoided."

---

Input:
"The reason this happened is because of misconfiguration."

Output:
"This happened because of misconfiguration."

---

## Explanation Style

Bad:
"A source map maps compiled code to source code."

Good:
"A source map is like a translation dictionary between compiled code and your original code."

---

Bad:
"The system registers the executable in PATH."

Good:
"Your system remembers where it lives, so when you run the command, it just works."

---

## Insight / Punchlines

Input:
"This shows developers should be careful."

Output:
"AI didn't cause this. We just stopped checking."

---

Input:
"This was a small mistake."

Output:
"A small mistake with big consequences."

---

## Formatting Patterns

Dense:
"This process includes building the code, checking issues, verifying dependencies, and publishing."

Improved:
"This process looks like this:

- build the code  
- check issues  
- verify dependencies  
- publish"

---

## Anti-Patterns

Avoid:

- overly formal language
  "This highlights the importance of..."

- generic AI phrasing
  "In today's fast-paced world..."

- unnecessary repetition
  "This is very important and crucial and critical"

- over-explaining simple ideas

- rewriting everything when small edits would work
